# Session Handover

セッションの作業内容を要約し、次のセッションのClaudeがスムーズに引き継げる
HANDOVER.mdを作成する。

## 手順

### Step 1: セッション内容の振り返り

会話全体を振り返り、以下の情報を整理する：

- 何を達成したか（完了したタスク）
- 何を決定したか（設計判断、技術選定など）
- どんな問題に遭遇したか（バグ、落とし穴、予想外の挙動）
- 何を学んだか（教訓、知見）
- 何が未完了か（残タスク、ブロッカー）

### Step 2: HANDOVER.md の作成

プロジェクトルートに `HANDOVER.md` を作成する。

- 既存ファイルがある場合は内容を表示し、**上書き/追記/キャンセル**の選択を
  ユーザーに求める
- 日時は `date '+%Y-%m-%d %H:%M'` で取得する
- ブランチ名は `git branch --show-current` で取得する
- コード状態は `git status` と `git diff --stat` で確認する
- 該当なしのセクションは省略してよい

```markdown
# Session Handover

**日時**: YYYY-MM-DD HH:MM
**ブランチ**: <現在のブランチ名>

## 完了したタスク

- タスク1の説明
- タスク2の説明

## 決定事項

- **決定内容**: 理由・背景
- **決定内容**: 理由・背景

## 遭遇した問題・落とし穴

- **問題**: 原因と対処法
- **問題**: 原因と対処法

## 学んだ教訓

- 教訓1
- 教訓2

## 未完了タスク・次のアクション

- [ ] タスク1の説明（優先度: 高/中/低）
- [ ] タスク2の説明（優先度: 高/中/低）

## 現在のコード状態

- コミット済み/未コミットの変更の概要
- テストの状態
- 既知の問題

## コンテキスト（次のセッションへの注意点）

- 作業中に気づいたが手を付けなかったこと
- 次のセッションで最初に確認すべきこと
```

### Step 3: SKILL / CLAUDE.md の改善提案

セッション中の作業から得た知見をもとに、以下の改善を提案する：

1. **CLAUDE.md の改善**:
   - 新たに判明したプロジェクト固有のルールやパターン
   - 追加すべき開発コマンドや注意事項
   - 既存の記述で修正が必要な箇所

2. **スキルの改善**:
   - 既存スキルで不足している機能や説明
   - 新しいスキルとして切り出すべきパターン

3. **ルールファイルの改善** (`.claude/rules/`):
   - 追加すべきルールやガイドライン

改善提案はユーザーに確認の上、承認されたもののみ反映する。

## 次のセッションでの使い方

次のセッション開始時に以下のようにClaudeに指示する：

```text
HANDOVER.md を読んで前回の作業を確認してください
```

引き継ぎ完了後、HANDOVER.mdは不要になるため削除してよい。

## 注意事項

- HANDOVER.mdは一時的な引き継ぎ文書であり、永続的なドキュメントではない
- プロジェクトの `.gitignore` にHANDOVER.mdを追加することを推奨
- 機密情報（パスワード、トークン等）は絶対に含めない
