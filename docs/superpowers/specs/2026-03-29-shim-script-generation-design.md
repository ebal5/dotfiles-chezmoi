# uvx/bunx Shim Script Auto-Generation

## Summary

`chezmoi apply` 時に、定義ファイルに基づいて `uvx` / `bunx` のラッパースクリプト（shim）を
`~/.scripts/` に自動生成する仕組み。エイリアスでは非対話環境（Claude Code等）から
呼び出せない問題を解決する。

## Background

- シェルエイリアス（`alias ruff="uvx ruff"`）は対話シェルでのみ有効
- Claude Code などの非対話環境ではエイリアスが展開されず、ツールが見つからない
- PATH に含まれる実行可能スクリプトであれば、どの環境からも呼び出せる

## File Structure

| File (chezmoi source) | Deployed to | Role |
| --- | --- | --- |
| `dot_config/shim-definitions` | `~/.config/shim-definitions` | Tool definition file |
| `run_after_generate_shims.sh.tmpl` | (executed only) | Shim generator script |

## Definition File Format

Location: `dot_config/shim-definitions`

```txt
# runner:package[:alias]
# Lines starting with # and empty lines are ignored
# When package contains @version, alias is required

uvx:ruff
uvx:bandit
uvx:mypy
uvx:ty
bunx:markdownlint-cli2
uvx:some-tool@1.2.3:st
```

### Parse Rules

- Delimiter: `:`
- Field 1: Runner (`uvx`, `bunx`)
- Field 2: Package name (may include `@version`)
- Field 3 (optional): Alias (command name override)
- If field 2 contains `@` and field 3 is absent: error, skip with warning
- If field 3 is absent: command name = field 2

## Generated Shim Format

Path: `~/.scripts/<command-name>`

```bash
#!/bin/sh
# AUTO-GENERATED by chezmoi shim — do not edit
exec <runner> <package> "$@"
```

- Marker comment `AUTO-GENERATED by chezmoi shim` identifies managed shims
- `exec` replaces the shell process (no extra process overhead)
- File permission: executable (755)

## run_after_ Script Behavior

File: `run_after_generate_shims.sh.tmpl`

Wrapped in `{{ if ne .chezmoi.os "windows" }}...{{ end }}` to skip on Windows.

### Processing Flow

1. Read `~/.config/shim-definitions`
2. Parse each line, validate format
   - Skip comments (`#`) and empty lines
   - Error on `@` in package without alias: print warning to stderr, skip
3. For each valid entry:
   - Generate shim content
   - Compare with existing file at `~/.scripts/<command>`
   - Write only if content differs or file doesn't exist
   - `chmod +x` the file
4. Scan `~/.scripts/` for files containing `AUTO-GENERATED by chezmoi shim` marker
5. Delete any marker-bearing file whose command name is not in current definitions

### Error Handling

- Missing definition file: exit silently (no shims to generate)
- Invalid line format: warn to stderr, skip, continue processing
- `~/.scripts/` directory missing: create it

## Migration

Remove the following alias lines from `dot_config/sh-like-aliases`:

```bash
alias ruff="uvx ruff"
alias bandit="uvx bandit"
alias mypy="uvx mypy"
alias ty="uvx ty"
```

These are replaced by the generated shim scripts.
