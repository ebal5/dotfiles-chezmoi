# ドキュメント作成ガイドライン

## Markdownスタイル

Markdownlintルールに準拠したドキュメント作成を推奨:

- **MD031**: フェンスコードブロック（``` ）の前後には空行を入れる
- **コードブロックの書式**:

  ```markdown
  テキスト

  ```bash
  コード内容
  ```

  続きのテキスト
  ```

- **MD040**: フェンスコードブロックには言語指定が必要

  ```bash
  # フェンスコードブロックの正しい形式
  echo "hello world"
  ```

- **見出し**: 一貫したレベル構造を維持
- **リスト**: インデントと記号の統一

## ドキュメント構成

- **概要**: プロジェクト/機能の簡潔な説明
- **使用方法**: 具体的なコマンド例と説明
- **設定**: 設定ファイルの場所と重要なオプション
- **トラブルシューティング**: よくある問題と解決方法
